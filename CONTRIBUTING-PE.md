## MANDATORY: No Direct Pushes to Main

**Rule:** All work must go through Pull Requests

## Required Workflow

1. Create branch: `git checkout -b feature/your-feature-name`
2. Open PR **within 24 hours** (can be draft)
3. Get **at least 1 teammate review**
4. Check all DoD boxes in PR template
5. Merge only after approval

## Violations

If you push direct to main, you must:

- Revert the commit
- Redo via proper PR process
- Document in team retro
